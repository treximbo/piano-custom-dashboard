Information & Resources

This page documents how to set up and use the app, including the browser extension for capturing Piano Composer tokens.

Quick Start

  1. Optional: Download the Piano Composer Extension (ZIP) and unzip it.
  2. Open chrome://extensions → enable Developer Mode → Load unpacked → select the unzipped extension folder.
    Note: Browsers often block direct links to chrome:// pages. Copy the URL and paste it into the address bar.
  3. Open the Piano dashboard in a tab: Piano Dashboard.
  4. In the dashboard, open an experience and click its Conversion Report so a composer conversion API request is made; the extension will capture its Bearer token.
  5. Return here and click Connect via Browser Extension. The status should read: “Connected to Piano (Composer token received).”
  6. Manual tab: paste or use the captured token, choose brand, expId, and date range, then Fetch.
  7. Automatic tab: choose brand and click Load Experiences; then click an experience to view its data.

Troubleshooting Token Capture

  • If you see “Token invalid or expired…”, the Composer token likely expired. Click Connect again while the Piano dashboard tab is open.
  • If “No response from extension…” shows: ensure the extension is installed, Site access is allowed for this site, and try again.
  • You can always paste a fresh Bearer token manually in the Manual tab as a fallback.

Environment Variables (Server)

  • PIANO_API_TOKEN or PIANO_API_TOKEN_<AID>: used by Automatic → Experiences endpoint.
  • TRENDS_CACHE_TTL: seconds to cache trend slices.
  • Do not store user Composer tokens on the server; tokens are kept in the browser only.

Security Notes

  • Keep this app behind an auth gate if exposing publicly.
  • Composer tokens expire regularly; don’t hardcode them.
  • Only install the extension where needed; review its host permissions.
